在这段代码中，第三个结果输出为 `-1` 是由两个关键的逻辑误区导致的：**“地址的地址”与“栈内存的增长方向”**。

### 1. 核心错误：`&p_end` vs `p_end`

在 C 语言中，这两个表达式代表完全不同的内存位置：

* **`p_end`**：存储在指针变量里的**内容**（即数组元素 `pressure[60]` 的内存地址）。
* **`&p_end`**：指针变量**本身在内存（栈）中的地址**（即存放这个“路标”的柜子本身的编号）。

当你执行 `&p_end - &p_start` 时，你计算的是**存放这两个指针变量的“柜子”**之间的距离，而不是它们指向的数组元素之间的距离。

### 2. 为什么结果是 `-1`？

这涉及到计算机**栈（Stack）**的布局逻辑：

* **定义顺序**：你在代码中先定义了 `p_start`，随后定义了 `p_end`。
* **增长方向**：在大多数系统（如 Windows/x64）中，栈是**向低地址增长**的。这意味着先定义的变量位于较高的地址，后定义的变量位于较低的地址。
* **步长单位**：指针相减的单位是**所指向类型的大小**。由于 `&p_end` 的类型是 `double **`（指向指针的指针），其步长是一个指针的大小（在 64 位系统下为 8 字节）。
* **计算过程**：
假设 `p_start` 存储在 `0x...10` 地址，`p_end` 紧挨着它存储在 `0x...08` 地址：
`(&p_end - &p_start)` = `(低地址 - 高地址) / 8 字节` = `(-8 字节) / 8 字节` = **`-1`**。

### 3. 关于“字节”与“元素个数”的混淆

你的代码中还存在关于单位的第二个误解：

* **`p_end - p_start`**：计算结果是 **50**。这是因为 C 语言的指针算术会自动除以类型的大小（`sizeof(double)`）。它返回的是**元素的个数**，而不是字节数。
* 如果你想要得到**字节数**，必须显式地将指针转换为字节单位（`char *`）或者手动乘以大小：

```c
// 方法 A：转换为字节指针后再相减
long byte_distance = (char *)p_end - (char *)p_start; 
printf("Distance in bytes: %ld\n", byte_distance); // 输出 400

// 方法 B：元素个数乘以类型大小
long byte_distance_2 = (p_end - p_start) * sizeof(double);
printf("Distance in bytes: %ld\n", byte_distance_2); // 输出 400

```

### 修改建议

为了达到你研究“地址算术”的目的，建议修改代码如下：

1. **移除变量前的 `&` 符号**：这样你操作的才是数组的地址，而不是指针变量自己的地址。
2. **区分逻辑距离与物理距离**：理解 `50`（元素个数）与 `400`（字节数）的转换关系。

```c
// 正确的逻辑距离计算（元素个数）
long elements = p_end - p_start; // 结果为 50

// 正确的物理距离计算（字节数）
```
