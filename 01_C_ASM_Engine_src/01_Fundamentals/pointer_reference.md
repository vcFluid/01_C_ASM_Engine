### 1. 为什么你没看到“预期的结果”？

在现代操作系统中，程序运行在**虚拟地址空间**中。当你试图访问地址 `10` 时，会发生以下连锁反应：

1. **物理隔离**：地址 `0` 到 `4096`（通常是一个内存页的大小）被操作系统标记为**“禁止访问区”**。
2. **MMU 介入**：当你执行 `*p`（其中 `p=10`）时，CPU 的内存管理单元（MMU）会发现这个地址没有访问权限。
3. **系统击毙**：操作系统会立即向你的程序发送一个 `SIGSEGV`（段错误）信号。

**结果**：你的程序会瞬间消失（闪退），或者报错“Segmentation Fault”，而不会继续执行后面的 `printf`。你无法看到“变量未初始化”的样子，因为你连看一眼的权利都没有。

### 2. 代码中的三个物理“禁区”

#### A. 向“虚无”写数据

```c
int *p; 
*p = a; // 此时 p 里的值是随机垃圾，比如 0x7FF...

```

* **物理本质**：你正试图往一个随机的、不属于你的“柜子”里塞东西。这可能覆盖掉其他变量，甚至导致系统底层崩溃。

#### B. 访问地址 `10`

```c
p = a; // p 变成了 10
printf("%d", p); // 这里打印的是 10，没有崩溃
*p; // 只要你敢写 *p，程序必崩

```

* **物理本质**：在计算机底层，地址 `10` 位于**零页（Zero Page）**附近。这是专门留出来捕捉 NULL 指针错误的陷阱区。

#### C. 类型混淆

```c
*p = &a; 

```

* **物理本质**：`*p` 期待的是一个整数（4 字节），而你塞入的是一个地址（8 字节）。这会导致**内存溢出（Overflow）**，多出来的 4 个字节会像溢出的流体一样，淹没相邻的内存格点。
